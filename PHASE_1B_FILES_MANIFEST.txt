================================================================================
DESIGN DECISIONS
================================================================================

1. INTERFACE SEGREGATION
   ✓ Each domain has its own interface (IReportConfig, etc.)
   ✓ Services only depend on what they need
   ✓ Mobile API never sees admin config keys
   ✓ Automation jobs never need admin settings

2. UNIFIED SOURCE OF TRUTH
   ✓ All config keys stored in GlobalConfig table
   ✓ GlobalConfigService manages persistence and caching
   ✓ Providers delegate to GlobalConfigService
   ✓ No code duplication

3. BACKWARD COMPATIBILITY
   ✓ Existing getAll() and update() methods still work
   ✓ Old code can continue using GlobalConfigService
   ✓ New code can use segregated providers
   ✓ Gradual migration path

4. SECURITY THROUGH SEGREGATION
   ✓ Mobile app cannot see session timeouts
   ✓ Automation jobs cannot see admin settings
   ✓ Least privilege principle applied
   ✓ HTTP endpoints enforce access control

5. TESTABILITY
   ✓ Each provider is independently testable
   ✓ Easy to mock interface methods
   ✓ No circular dependencies
   ✓ Clear separation of concerns
